# Case study library

Source of truth for every proof point used in outreach. One markdown file per case study,
copied from `_template.md`. The drafting skill picks the most relevant one per lead; the
fact-check skill verifies claims against these files word for word.

Rules:
- Only verified, client-approved facts and numbers. If a number is an estimate, don't include it.
- `icp_fit` tells the drafter which audience a case study can be shown to.
- `public_url` is optional. Once the case study is live on hellopolymer.com, add the URL
  and emails can link to it. Until then it's citable in text but not linkable.

Currently empty except the template. This is the single biggest blocker to strong
personalization — even 2 short ones (one startup, one church) changes draft quality a lot.
